🎯 Project Overview

A Data Cleaning OpenEnv Environment where AI agents learn to clean dirty tabular data. The environment generates corrupted employee datasets with realistic errors — typos, wrong formats, missing values, duplicates, and logical inconsistencies — and an LLM agent must identify and fix them step-by-step through a RESTful API.

Why Data Cleaning?Details
Real-world impactData scientists spend ~80% of their time on data cleaning
Measurable progressEach cell fixed = immediate incremental reward signal
Deterministic gradingGround-truth comparison ensures fair, reproducible evaluation
LLM-native designCSV text + JSON actions work naturally with language models

🏗️ System Architecture

LLM Agent (from openai import OpenAI) │ │ HTTP ── POST /reset, POST /step, GET /state, GET /grade ▼ FastAPI Server (HuggingFace Space — port 7860) │ ├── DataGenerator → Seeded dirty data + error manifest (ground truth) ├── Environment → Action validation + reward computation └── TaskGrader → Deterministic scoring: errors_fixed / total_errors

💰 Reward Function

Continuous reward signal providing meaningful partial progress at every step:

EventRewardPurpose
Correctly fix an error+1.0 / total_errorsIncremental partial credit
Wrong fix attempt−0.05Penalize incorrect guesses
Delete valid row−0.20Penalize destructive actions
No-effect action−0.05Penalize wasted steps
